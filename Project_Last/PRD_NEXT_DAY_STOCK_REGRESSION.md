# 삼성전자 익일 시가·종가 예측 회귀 모델 PRD

## 1. 문서 개요

- 프로젝트명: 삼성전자 익일 시가·종가 예측 모델
- 대상 종목: 삼성전자(005930)
- 입력 데이터: `005930_2016to2026_수정됨.txt`
- 원본 데이터 범위: 2016-06-01 ~ 2026-06-01
- 모델 사용 범위: 2018-06-01 ~ 2026-06-01
- 문서 목적: 회귀 모델 구현 전에 목표, 데이터 처리, 검증 방식 및 완료 조건을 정의한다.

## 2. 배경

현재 CSV에는 일별 OHLCV 데이터와 다음 기술 지표가 포함되어 있다.

- 날짜
- 종가, 시가, 고가, 저가
- 거래량
- 변동률
- RSI(14)
- MACD(12, 26)
- MACD Signal(9)
- MACD Histogram
- 외국인소진율

이 데이터를 이용하여 특정 거래일 장 종료 시점까지 알 수 있는 정보만으로 다음 거래일의 시가와 종가를 추정하는 지도학습 회귀 모델을 개발한다.

본 모델은 연구 및 분석용이며 실제 투자 수익을 보장하는 매매 시스템을 목표로 하지 않는다.

## 3. 문제 정의

### 3.1 예측 시점

거래일 `t`의 장이 종료된 직후 예측을 수행한다고 가정한다.

### 3.2 입력

거래일 `t` 및 그 이전에 확정된 데이터만 사용한다.

### 3.3 목표값

다음 캘린더 날짜가 아닌 다음 실제 거래일 `t+1`의 값을 예측한다.

- `target_next_open`: 다음 거래일 시가
- `target_next_close`: 다음 거래일 종가

### 3.4 기본 예측 형태

가격 자체보다 전일 종가 대비 수익률을 우선 목표로 사용한다.

- 익일 시가 수익률: `next_open / current_close - 1`
- 익일 종가 수익률: `next_close / current_close - 1`

모델의 예측 수익률은 현재 종가를 이용해 원 단위 예상 가격으로 복원한다. 이 방식은 장기간의 가격 수준 변화와 주식분할 영향을 가격 자체를 직접 예측하는 방식보다 줄여 준다.

## 4. 목표와 비목표

### 4.1 목표

- 익일 시가와 종가를 각각 예측하는 재현 가능한 학습 파이프라인 구축
- 단순 기준 모델보다 우수한 성능 확보
- 시계열 순서를 지키는 검증으로 미래 데이터 누수 방지
- 예측값, 실제값, 오차 및 모델 성능을 파일로 저장
- 동일 데이터와 설정에서 동일 결과가 나오도록 랜덤 시드 고정

### 4.2 비목표

- 장중 고가·저가 또는 분봉 가격 예측
- 자동 주문 및 증권사 API 연동
- 거래 비용을 포함한 완전한 매매 전략 구축
- 뉴스, 공시, 환율, 지수 등 외부 데이터 사용
- 예측값을 투자 권유로 제공하는 기능

## 5. 사용자 시나리오

분석자는 정제된 최신 일별 데이터를 준비한 뒤 학습 또는 예측 명령을 실행한다.

시스템은 다음 작업을 수행한다.

1. 원본 데이터 검증 및 정제
2. 과거에서 미래 순서로 정렬
3. 파생 특성과 익일 목표값 생성
4. 시간 순서 기반 학습 및 검증
5. 기준 모델과 후보 모델의 성능 비교
6. 최종 모델, 설정, 평가 결과 저장
7. 최신 행을 기준으로 다음 거래일 시가·종가 예상치 출력

## 6. 데이터 요구사항

### 6.1 필수 열

| 구분 | 열 |
|---|---|
| 식별 | 날짜 |
| 가격 | 시가, 고가, 저가, 종가 |
| 거래 | 거래량, 변동률 |
| 지표 | RSI(14), MACD, MACD Signal, MACD Histogram |

### 6.2 정렬

원본 CSV는 최신 날짜부터 과거 날짜 순으로 저장되어 있다. 모든 특성 및 목표값 계산 전에 날짜 오름차순으로 정렬해야 한다.

### 6.3 숫자 변환

- 가격의 천 단위 쉼표 제거
- 변동률의 `%` 제거 후 실수 변환
- 거래량의 `K`, `M`, `B` 접미사를 실제 수치로 변환
- 날짜 내부 공백 제거 후 날짜형으로 변환

### 6.4 데이터 품질 검증

현재 데이터에는 다음과 같은 이상 징후가 확인되었다.

- 거래량 결측 행
- 휴장일 또는 비거래일로 보이는 행
- `4,900.00%`, `-97%` 수준의 비정상 변동률
- 인접 행과 가격 단위가 약 50배 차이 나는 행
- 2018년 액면분할 전후 가격 체계가 혼재했을 가능성

액면분할 이전 구간의 영향을 제거하기 위해 2018-06-01 이전 데이터는 모델 입력에서 전부 제외한다.

학습 전에 아래 규칙을 적용한다.

1. 날짜 중복 여부 검사
2. 한국거래소 실제 거래일 여부 또는 거래량 유효성 검사
3. 시가·고가·저가·종가의 논리 관계 검사
   - 고가 ≥ 시가와 종가
   - 저가 ≤ 시가와 종가
   - 모든 가격 > 0
4. 비정상 변동률 및 가격 배율 급변 행 탐지
5. 액면분할 기준으로 수정주가 일관성 확인
6. 제거 또는 보정된 행과 사유를 별도 로그에 기록

이상 행을 단순히 임계값으로 삭제하기 전에 원본 수집 오류인지 기업행동에 따른 정상 변화인지 구분한다.

## 7. 특성 설계

### 7.1 기본 특성

- 당일 시가, 고가, 저가, 종가
- 거래량
- 변동률
- RSI(14)
- MACD(12, 26)
- MACD Signal(9)
- MACD Histogram

가격 원값은 비정상성 문제를 줄이기 위해 비율 또는 수익률 형태로 변환하는 것을 원칙으로 한다.

### 7.2 파생 특성

- 일간 수익률: 1일, 2일, 3일, 5일, 10일, 20일
- 이동평균: 5일, 10일, 20일, 60일
- 종가와 각 이동평균의 거리
- 이동 표준편차: 5일, 10일, 20일
- 거래량 이동평균 및 거래량 비율
- 장중 수익률: `close / open - 1`
- 갭: `open / previous_close - 1`
- 일중 가격 범위: `(high - low) / close`
- 고가·저가 대비 종가 위치
- RSI와 MACD의 1일 변화량
- 요일, 월 등의 달력 특성

### 7.3 결측치

롤링 특성 계산으로 초기에 발생하는 결측치는 해당 행을 학습 데이터에서 제외한다. 미래 값으로 채우거나 전체 기간 통계로 채우지 않는다.

### 7.4 데이터 누수 방지

- 모든 입력 특성은 예측 시점에 이미 확정된 값이어야 한다.
- `t+1`의 거래량, 고가, 저가 또는 기술 지표를 입력에 포함하지 않는다.
- 스케일러와 전처리기는 학습 구간에만 적합한다.
- 전체 데이터로 이상치 경계나 통계값을 계산하지 않는다.
- 목표값 생성 후 마지막 행은 실제 목표가 없으므로 예측 전용 행으로 분리한다.

## 8. 모델 설계

### 8.1 기준 모델

후보 모델이 실제로 가치를 추가하는지 확인하기 위해 다음 기준을 둔다.

- Naive Open: 다음 시가 = 현재 종가
- Naive Close: 다음 종가 = 현재 종가
- 최근 N일 평균 수익률 기반 예측
- 선형 회귀 또는 Ridge 회귀

### 8.2 1차 후보 모델

- Ridge / Elastic Net
- Random Forest Regressor
- HistGradientBoostingRegressor
- XGBoost 또는 LightGBM은 의존성 추가가 허용될 때 선택적으로 검토

초기 버전은 해석성과 재현성이 높은 Ridge와 비선형 트리 부스팅 모델을 비교한다.

### 8.3 다중 목표 처리

아래 두 방식을 비교할 수 있도록 구성한다.

- 시가와 종가 모델을 각각 학습
- 하나의 다중 출력 회귀 모델로 동시 학습

초기 구현은 오류 분석이 쉬운 개별 모델 방식을 기본으로 한다.

## 9. 학습 및 검증 전략

무작위 train/test 분할은 사용하지 않는다.

### 9.1 홀드아웃 분할

날짜 기준으로 다음과 같이 분할한다.

- Train: 초기 데이터부터 전체의 80%
- Validation: 다음 10%
- Test: 가장 최근 10%
- Train/Validation 및 Validation/Test 경계에서 익일 목표가 다음 구간을 참조하는 마지막 행을 1개씩 제거한다.

정확한 경계 날짜와 각 구간의 행 수는 데이터 정제 후 설정 파일과 결과 보고서에 기록한다.

### 9.2 워크포워드 검증

하이퍼파라미터 비교에는 확장 윈도우 방식의 `TimeSeriesSplit`을 사용한다.

```text
Fold 1: [Train -------------][Validation]
Fold 2: [Train ----------------------][Validation]
Fold 3: [Train -------------------------------][Validation]
```

최종 테스트 구간은 모델과 설정을 확정할 때까지 사용하지 않는다.

## 10. 평가 지표

### 10.1 핵심 지표

- MAE: 원 단위 평균 절대 오차
- RMSE: 큰 오차에 민감한 원 단위 오차
- MAPE 또는 sMAPE: 가격 수준 대비 상대 오차
- 수익률 MAE
- 방향 정확도: 실제 상승·하락 방향 일치 비율

시가와 종가 결과를 각각 보고한다.

### 10.2 추가 진단

- 실제값과 예측값 시계열 그래프
- 잔차 분포
- 월별 또는 연도별 오차
- 상승장과 하락장 구간별 오차
- 큰 갭 발생일의 성능
- 특성 중요도 또는 선형 모델 계수

### 10.3 성공 기준

최종 후보 모델은 독립 테스트 구간에서 다음 조건을 만족해야 한다.

- 시가 MAE가 Naive Open 기준보다 개선
- 종가 MAE가 Naive Close 기준보다 개선
- 일부 기간에만 성능이 집중되지 않고 워크포워드 폴드 전반에서 안정적
- 학습과 추론을 오류 없이 재실행 가능

방향 정확도 50% 초과는 참고 지표로 보되, 통계적 유의성과 거래 비용 검토 없이 수익성을 의미한다고 해석하지 않는다.

## 11. 기능 요구사항

### FR-1. 데이터 준비

시스템은 원본 CSV를 읽고 날짜와 숫자 형식을 정규화해야 한다.

### FR-2. 데이터 검증

시스템은 결측값, 중복 날짜, OHLC 오류, 비정상 가격 변화 및 거래량 오류를 탐지해야 한다.

### FR-3. 특성 생성

시스템은 설정된 기간의 수익률, 이동평균, 변동성 및 거래량 특성을 생성해야 한다.

### FR-4. 목표 생성

시스템은 각 행에 다음 거래일의 시가·종가 수익률과 실제 가격 목표를 생성해야 한다.

### FR-5. 모델 학습

시스템은 기준 모델과 복수 후보 모델을 동일한 분할에서 학습해야 한다.

### FR-6. 평가

시스템은 모델별·목표별 성능을 표 형태로 저장해야 한다.

### FR-7. 최신 예측

시스템은 목표값이 없는 최신 행을 이용해 다음 거래일 예상 시가와 종가를 출력해야 한다.

### FR-8. 결과 저장

최소한 다음 산출물을 저장해야 한다.

- 정제 데이터
- 데이터 품질 보고서
- 학습된 모델
- 전처리 파이프라인
- 모델 설정
- 테스트 예측 결과
- 성능 지표
- 최신 예측 결과

## 14. 개발 단계

### Phase 1. 데이터 감사 및 정제

- 비정상 행의 원인 확인
- 수정주가 기준 통일
- 거래량 파싱
- 품질 보고서 생성

완료 조건: 모델 입력에 사용할 유효 거래일 데이터셋과 제외 행 목록이 생성된다.

### Phase 2. 기준 파이프라인

- 목표값과 기본 특성 생성
- 시간 순서 분할
- Naive 및 Ridge 모델 구현
- 기본 평가 보고서 생성

완료 조건: 처음부터 끝까지 한 번에 실행되는 기준 파이프라인이 존재한다.

### Phase 3. 후보 모델 비교

- 트리 기반 모델 추가
- 워크포워드 검증
- 제한적인 하이퍼파라미터 탐색
- 모델별 성능 및 안정성 비교

완료 조건: 독립 테스트 전에 최종 후보와 설정이 결정된다.

### Phase 4. 최종 평가 및 예측

- 고정된 최종 모델로 테스트 구간 평가
- 최신 행 기반 다음 거래일 예측
- 모델 카드와 한계 정리

완료 조건: 결과가 파일로 저장되고 동일 명령으로 재현된다.

## 15. 테스트 및 인수 기준

- 날짜가 오름차순으로 처리된다.
- 입력 행 `t`의 목표가 실제 다음 거래일 `t+1`과 일치한다.
- 주말과 휴장일을 목표 행으로 인위적으로 생성하지 않는다.
- 전처리기가 학습 데이터에만 적합된다.
- 원본 CSV는 변경되지 않는다.
- 비정상 행은 무시되지 않고 보고서에 기록된다.
- 기준 모델과 후보 모델이 동일한 테스트 행에서 비교된다.
- 학습 결과에 사용한 데이터 기간, 특성 목록, 모델 설정 및 랜덤 시드가 기록된다.
- 최신 데이터에 대해 시가·종가 예측값과 기준일이 명확하게 출력된다.
- 단위 테스트 및 전체 파이프라인 실행이 성공한다.

## 16. 주요 위험과 대응

| 위험 | 영향 | 대응 |
|---|---|---|
| 휴장일·오염 데이터 | 지표 및 목표값 왜곡 | 실제 거래일 검증과 제외 로그 |
| 액면분할 전후 단위 불일치 | 극단적 수익률 발생 | 수정주가 기준 통일 |
| 데이터 누수 | 과도하게 좋은 평가 | 시간 순서 분할과 파이프라인 사용 |
| 데이터 수 부족 | 과적합 | 단순 모델 우선, 제한된 특성 및 검증 |
| 시장 구조 변화 | 최근 구간 성능 저하 | 워크포워드 및 기간별 성능 분석 |
| 원 가격 직접 예측 | 장기 추세 의존 | 수익률 목표 학습 후 가격 복원 |
| 단일 종목 의존 | 일반화 제한 | 결과의 적용 범위를 삼성전자로 명시 |

## 17. 향후 확장

1차 모델이 기준 성능을 충족한 이후에만 다음 항목을 검토한다.

- KOSPI 및 반도체 지수
- 원/달러 환율
- 외국인·기관 수급
- 공시 및 뉴스 감성
- 미국 반도체 관련 종목과 지수
- 예측 구간 또는 불확실성 추정
- 백테스트와 거래 비용 반영

## 18. 결정 사항 요약

- 예측 대상은 다음 실제 거래일의 시가와 종가다.
- 입력은 당일 장 종료 시점까지 확정된 정보만 사용한다.
- 기본 목표는 전일 종가 대비 익일 수익률이며, 최종 출력은 원 단위 가격으로 복원한다.
- 데이터는 무작위로 나누지 않고 시간 순서로 분할한다.
- 오염 데이터와 액면분할 문제를 해결한 후 모델링한다.
- Naive 모델과 Ridge를 기준으로 삼고 트리 부스팅 모델을 비교한다.
- 성능은 가격 오차, 수익률 오차 및 방향 정확도를 함께 평가한다.
